## 2. Design System

### 2.1 Color Tokens

```ts
// tailwind.config.ts
colors: {
  espresso: {
    950: '#1C0F07',  // teks heading utama
    800: '#3D2010',  // teks body, sidebar dark
  },
  kopi: {
    600: '#7C4A22',  // primary brand / CTA button
    400: '#B07040',  // hover states, link
    100: '#F3E8DC',  // bg highlight ringan
  },
  kakao: {
    200: '#E8CEAA',  // card border, divider
    100: '#F5EDDF',  // tabel stripe
  },
  krem: {
    50:  '#FAF5EC',  // background halaman
  },
  sawah: {
    700: '#3D6B2C',  // accent hijau / badge status / CTA sekunder
    400: '#78A85A',  // success, icon accent
    100: '#E8F4E2',  // bg badge hijau
  },
  tanah: {
    500: '#C0562A',  // alert/warning (tanah merah NTT)
    100: '#FAE8E0',  // bg badge merah
  },
}
```

### 2.2 Typography

- **Heading:** Plus Jakarta Sans, `font-semibold`, scale: `text-4xl` → `text-6xl` (hero), `text-2xl` → `text-3xl` (section)
- **Body:** Inter, `font-normal`, `text-base` (15px), `leading-relaxed` (1.6)
- **Angka statistik:** JetBrains Mono atau Inter numeric, `font-bold`, `text-3xl`+

### 2.3 Styling Rules

- **Border radius:** `rounded-xl` (card), `rounded-lg` (button/input), `rounded-full` (badge/pill)
- **Shadow:** `shadow-sm` (card default), `shadow-md` (hover), `shadow-lg` (modal/dialog)
- **Transisi:** `transition-all duration-200` sebagai standar
- **Tone visual:** Hangat & approachable — tidak ada elemen keras/angular

## 4. Layout Global

### Navbar
- **Desktop:** Logo (kiri) + link horizontal tengah: Beranda · Profil Desa ▾ · Data Desa ▾ · Pemerintahan · Lembaga Desa · Berita · Layanan
- **Mobile:** Logo + hamburger → slide-down overlay fullwidth
- Style: `bg-krem-50/80 backdrop-blur-md`, border-bottom `kakao-200`, sticky `top-0 z-50`
- Link aktif: `text-kopi-600 font-semibold`; hover: `text-kopi-400`
- Dropdown (Profil Desa & Data Desa): `bg-white shadow-lg rounded-xl` muncul on hover/focus

### Footer
3 kolom:
1. Info desa: nama, alamat, logo
2. Navigasi: semua link utama
3. Kontak: email, jam layanan

Style: `bg-espresso-800 text-krem-50`, link hover `text-kopi-400`

### PageHeader
Banner tipis di bawah navbar untuk halaman dalam (bukan Beranda):
```
bg: linear-gradient(kopi-600 → espresso-800)
Judul halaman: white, Plus Jakarta Sans semibold, text-3xl
Breadcrumb: white/70%, text-sm
```

## 5. Desain Per Halaman

### 5.1 Beranda (`/`)

**Hero Section**
- Full viewport height (`min-h-screen`)
- Background: `bg-gradient-to-br from-espresso-950 to-kopi-800` + overlay foto desa `opacity-40` (placeholder gradient sementara)
- Konten tengah (centered):
  - Label kecil: "Selamat Datang di" (krem/70%)
  - H1: "DESA SITA" (putih, text-5xl md:text-7xl, font-bold, tracking-wide)
  - Sub: "Kec. Rana Mese · Kab. Manggarai Timur · NTT" (krem/60%)
  - 2 CTA button: "Jelajahi Desa ↓" (bg kopi-600) + "Data Desa →" (outline putih)
  - Scroll indicator animasi bounce di bawah

**Statistik Snapshot**
- 4 StatCard horizontal (grid-cols-2 md:grid-cols-4)
- Data: Total Penduduk, Jumlah KK, Luas Wilayah, Jumlah Dusun
- Card: bg-white, shadow-sm, border kakao-200, rounded-xl

**Potensi Desa**
- 3 card: Pertanian, Perkebunan, Peternakan
- Icon Lucide besar (kopi-600), judul, deskripsi singkat, link "Lihat Detail →"
- bg: kopi-100, hover shadow-md

**Berita Terbaru**
- Grid 3 BeritaCard terbaru
- Tombol "Lihat Semua Berita →" di bawah

**CTA Pengaduan**
- Banner full-width, bg sawah-700
- Teks: "Ada masukan untuk desa?" + sub teks
- Tombol: "Kirim Pengaduan →" (bg-white text-sawah-700)

### 5.2 Data Desa — Sub-halaman

Pola konsisten untuk semua `/data-desa/*`:

1. **PageHeader** (gradient kopi)
2. **Card ringkasan** — angka-angka kunci (2–4 StatCard)
3. **Chart** — `<ResponsiveContainer width="100%" height={300}>`
4. **Tabel detail** — stripe kakao-100, header espresso-800/10, badge untuk nilai tertinggi
5. **DataUpdatedAt** — "Data terakhir diperbarui: — hari lalu" (placeholder)

Chart per sub-halaman:
| Halaman | Chart |
|---|---|
| Kelompok Umur | `BarChart` horizontal, warna kopi-600 |
| Jenis Kelamin | `PieChart` donut, kopi-600 vs sawah-700 |
| Pendidikan | `BarChart` vertikal, gradient kopi |
| Sarana Prasarana | Grouped card per kategori (tidak pakai chart) |
| Wilayah Administratif | StatCard grid saja |

### 5.3 Pemerintahan (`/pemerintahan`)

- **Struktur Organisasi:** visual tree card-based (CSS flexbox, bukan library khusus)
  - Level 1: Kepala Desa
  - Level 2: Sekretaris Desa
  - Level 3: Kaur-kaur & Kasi-kasi
  - Card: avatar placeholder (inisial nama, bg kopi-100) + jabatan + nama + pendidikan
- **Tabel BPD:** Nama | Jabatan | Pendidikan, style konsisten

### 5.4 Lembaga Desa (`/lembaga-desa`)

- Tab / filter pill per kategori: Kemasyarakatan · Ekonomi · Pendidikan · Keamanan
- Card grid per lembaga: nama, dasar hukum, jumlah pengurus, keterangan

### 5.5 Profil Desa

- `/profil-desa` — hub dengan card link ke sub-halaman
- `/profil-desa/sejarah` — timeline kepala desa + narasi sejarah
- `/profil-desa/wilayah` — batas wilayah (prosa), tabel komoditas, tabel peternakan, info iklim

### 5.6 Berita (`/berita` + `/berita/[slug]`)

**Daftar:**
- Filter pill: Semua · Berita · Pengumuman
- Grid 3 kolom desktop / 1 kolom mobile
- BeritaCard: cover img (next/image, aspect-video) + badge kategori + judul + ringkasan + tanggal

**Detail:**
- Cover image full-width (max-h-80, object-cover)
- Breadcrumb + badge kategori
- H1 judul, meta (penulis · tanggal · estimasi baca)
- Konten artikel: `prose prose-stone` (Tailwind Typography plugin)
- Berita lainnya (3 card) di bagian bawah

### 5.7 Layanan (`/layanan`)

- Info layanan operasional (jam, prosedur) — konten statis
- Card CTA ke form pengaduan

### 5.8 Form Pengaduan (`/layanan/pengaduan`)

- Card `max-w-2xl mx-auto`
- Field:
  - Nama (optional, placeholder "Anonim jika dikosongkan")
  - No. Kontak (optional)
  - Kategori (Select: Infrastruktur / Pelayanan / Sosial / Lainnya)
  - Isi Pengaduan (Textarea, minLength 20 dengan counter)
  - Catatan privasi (ikon gembok kecil, teks krem/muted)
- Button: "Kirim Pengaduan" bg-kopi-600 full-width
- States: idle → loading (spinner) → sukses (hijau sawah, pesan terima kasih) → error (tanah-500)
- Validasi: Zod schema client-side, pesan error per field
- Sementara tanpa integrasi: submit log ke console + tampilkan success state

### 5.9 Rencana Pengembangan (`/rencana-pengembangan`)

- Halaman statis
- Grid card "Segera Hadir": Layanan Surat Online, RPJM/RKP, LPPD, IDM
- Setiap card: ikon + nama + deskripsi singkat + badge "Segera Hadir" (tanah-500)

## 6. Komponen Shared & UI States

### Loading Skeleton
- Komponen `<Skeleton />` dari shadcn/ui
- Dibungkus `<Suspense fallback={<XxxSkeleton />}>` per section
- Warna: `bg-kakao-200 animate-pulse`
- Tersedia untuk: StatCard, BeritaCard, Chart, Tabel

### Empty State
```tsx
<EmptyState
  icon={<FileQuestion />}
  message="Belum ada data tersedia"
/>
```
Style: centered, icon espresso/30%, teks espresso/40%

### Error State
```tsx
<ErrorState
  message="Gagal memuat data"
  onRetry={() => {}}
/>
```
Style: border tanah-100, icon tanah-500, tombol outline

## 8. Responsivitas

| Breakpoint | Perilaku |
|---|---|
| `< 640px` (mobile) | 1 kolom, nav hamburger, font scale -1 step, chart height 220px |
| `640–1024px` (tablet) | 2 kolom card, nav hamburger, chart height 260px |
| `> 1024px` (desktop) | 3–4 kolom card, nav horizontal, chart height 300px |

Chart Recharts selalu dibungkus `<ResponsiveContainer width="100%" height={...}>`.
